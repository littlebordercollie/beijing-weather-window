# 雨窗｜北京目的地降雨决策地图

输入北京目的地、预计到达耗时和 3/5 公里观测半径，把公开参考台站、五个周边格点和未来两小时降雨窗口放在同一个界面里。

公网地址：部署后为 `https://littlebordercollie.github.io/beijing-weather-window/`

## 页面

- `index.html`：地图视图，可搜索地点、设备定位或直接点击地图选点。
- `list.html`：列表视图，展开台站、格点、更新时间、未来时次和全市水务日雨量背景。

## 当前数据口径

- 18 个参考台站的站号、坐标和海拔来自北京市地方标准 DB11/T 1643—2023。
- 实时天气与未来时次来自 Open-Meteo 格点产品，界面明确标注为“站旁格点”，不冒充台站原始观测。
- 北京市水务局公开日雨量响应当前包含 121 个站点行；因为响应没有经纬度，不进入 3/5 公里附近计数。
- 地点搜索和地图来自 OpenStreetMap / Nominatim。
- 页面是出行决策辅助，不是官方气象预报；强对流天气以官方预警为准。

## 当前结论

这个东西能做，但要拆成两个不同目标：

1. **北京观测站地图**：看不同观测网络的站点、当前值和历史值。
2. **出行降雨助手**：输入目的地和预计到达时间，回答“现在走会不会淋、雨大约什么时候走”。

本轮在已检索的北京官方、国家级公开数据平台、主流天气服务和开发者接口中，**没有找到一个公开网站同时满足以下四项**：

- 覆盖北京“所有”观测站；
- 提供每个站的实时多要素原始观测；
- 提供逐站、分钟或小时级历史曲线；
- 同时提供雷达移动和目的地停雨判断。

这里的“所有”本身也不是一个单一集合。北京至少同时存在国家级气象站、区域自动气象站、水务雨量站、塔基站和其他部门站网；站型不同，测量要素也不同。雨量站通常不能提供气压，网格预报也不能冒充真实站点。

现有公开产品已经把能力分散做掉了：

- 北京市水务局有 **121 个雨量站的 8 时至次日 8 时日雨量查询**，可按日期查询，但不是分钟实时，也没有气压、温度等多要素；
- 中国天气网有全国实况地图、降水/风/湿度和雷达页面，但没有提供“北京全部区域站逐站历史数据库”；
- 中国气象数据网提供北京国家站和融合实况产品，但公开国家站数据有时间粒度、滞后和注册限制；
- 深圳市气象局已经公开做出了非常接近目标形态的“自动站查询”，证明产品形态成立；
- 和风天气、彩云天气等商业服务已经提供任意坐标的分钟降水或雷达能力，但它们多为融合格点或商业数据，不是北京全部官方站的原始直播。

因此仍有产品空间，但价值不在于再造一张天气地图，而在于把观测证据转成行动建议：

> 我 40 分钟后到朝阳公园。现在出发会不会淋？如果等 10、20、30 分钟，哪个窗口更稳？

## 推荐开工方式

第一版不等待“全部官方区域站实时接口”授权，先做一个可用的目的地雨窗：

```text
地点或一句话
  -> 高德解析 POI 和路线
  -> 目的地 + 东西南北四个周边格点
  -> 分钟降水、实况、预警
  -> 过去两小时雷达动画
  -> 确定性程序计算雨窗和 ETA 风险
  -> AI 只负责解释结果与接受追问
```

与此同时申请区域自动站授权。拿到站号、坐标、站型和 5—10 分钟多要素数据后，再把周边“格点探针”替换或补充为真实站点。
