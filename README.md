# 雨窗｜北京目的地降雨决策地图

输入北京目的地、预计到达耗时和 3/5 公里观测半径，把公开参考台站、五个周边格点和覆盖行程的降雨窗口放在同一个界面里。

公网地址：<https://littlebordercollie.github.io/beijing-weather-window/>

## 页面

- `index.html`：地图视图，可搜索地点、设备定位或直接点击地图选点。
- `list.html`：列表视图，展开台站、格点、更新时间、未来时次和官方雨情页入口。

## 本地运行

```bash
npm ci
npm run dev
```

受支持的构建工具链固定为 Node.js 22（`>=22.12 <23`）和 npm 10。依赖解析以 `package-lock.json` 为准；复现构建请使用 `npm ci`，不要用会重新解析依赖的安装方式替代。

测试与构建：

```bash
npm test
npm run build
```

`npm run build` 会把生产文件写入 `docs/`。当前 GitHub Pages 从 `main:/docs` 发布，因此每次发布前必须在干净工作树中依次执行 `npm ci → npm test → npm run build`，并把与源码同步生成的 `docs/` 一起提交。自动化 Pages 工作流需要 GitHub `workflow` 授权后才能启用；不能把 legacy Pages 的“部署成功”当作测试已通过的证明。

## 当前数据口径

- 18 个参考台站的站号、坐标和海拔来自北京市地方标准 DB11/T 1643—2023。
- 实时天气与未来时次来自 Open-Meteo 格点产品，界面明确标注为“站旁格点”，不冒充台站原始观测。
- 北京市水务局日雨量仅保留官方页链接；数据接口的公开再使用权未确认，当前版本不调用、不缓存、不再发布其响应。
- 地点搜索使用基于 OpenStreetMap 数据的公共 Photon 服务，地图底图来自 OpenStreetMap / CARTO；公共地理编码服务只作为低流量、尽力而为的搜索源，不具备生产 SLA。
- 页面是出行决策辅助，不是官方气象预报；强对流天气以官方预警为准。

## 已知运行边界

- 任意地点在线搜索依赖第三方地理编码服务及其跨域、限流和缓存政策；常用地点本地索引、经纬度输入和地图点选是独立兜底。
- Open-Meteo 免费接口仅适用于非商业使用，没有可用性保证；商业或宣传用途必须换用合适的商业计划或已授权数据源。
- 北京市水务雨情接口不是本项目控制的版本化 API，当前不接入；获得明确授权后才考虑恢复。
- 站旁数值是格点估计，不是物理台站最后一次上传的原始读数；“下一时次”表示格点产品时次，不表示台站硬件必然在该时刻上报。
- 第三方服务恢复能力、浏览器跨域行为和完整端到端流程不能由当前确定性单元测试证明。

完整归因、许可和再使用边界见 [THIRD_PARTY_NOTICES.md](THIRD_PARTY_NOTICES.md)。

## 当前结论

这个东西能做，但要拆成两个不同目标：

1. **北京观测站地图**：看不同观测网络的站点、当前值和历史值。
2. **出行降雨助手**：输入目的地和预计到达时间，回答“现在走会不会淋、雨大约什么时候走”。

本轮在已检索的北京官方、国家级公开数据平台、主流天气服务和开发者接口中，**没有找到一个公开网站同时满足以下四项**：

- 覆盖北京“所有”观测站；
- 提供每个站的实时多要素原始观测；
- 提供逐站、分钟或小时级历史曲线；
- 同时提供雷达移动和目的地停雨判断。

这里的“所有”本身也不是一个单一集合。北京至少同时存在国家级气象站、区域自动气象站、水务雨量站、塔基站和其他部门站网；站型不同，测量要素也不同。雨量站通常不能提供气压，网格预报也不能冒充真实站点。

现有公开产品已经把能力分散做掉了：

- 北京市水务局有 **121 个雨量站的 8 时至次日 8 时日雨量查询**，可按日期查询，但不是分钟实时，也没有气压、温度等多要素；
- 中国天气网有全国实况地图、降水/风/湿度和雷达页面，但没有提供“北京全部区域站逐站历史数据库”；
- 中国气象数据网提供北京国家站和融合实况产品，但公开国家站数据有时间粒度、滞后和注册限制；
- 深圳市气象局已经公开做出了非常接近目标形态的“自动站查询”，证明产品形态成立；
- 和风天气、彩云天气等商业服务已经提供任意坐标的分钟降水或雷达能力，但它们多为融合格点或商业数据，不是北京全部官方站的原始直播。

因此仍有产品空间，但价值不在于再造一张天气地图，而在于把观测证据转成行动建议：

> 我 40 分钟后到朝阳公园。现在出发会不会淋？如果等 10、20、30 分钟，哪个窗口更稳？

## 推荐开工方式

第一版不等待“全部官方区域站实时接口”授权，先做一个可用的目的地雨窗：

```text
地点或一句话
  -> 高德解析 POI 和路线
  -> 目的地 + 东西南北四个周边格点
  -> 分钟降水、实况、预警
  -> 过去两小时雷达动画
  -> 确定性程序计算雨窗和 ETA 风险
  -> AI 只负责解释结果与接受追问
```

与此同时申请区域自动站授权。拿到站号、坐标、站型和 5—10 分钟多要素数据后，再把周边“格点探针”替换或补充为真实站点。
